# Documentation

The `docs/` directory keeps product, architecture, planning, and design
artifacts out of the repo root.

## Layout

- `architecture/` — product and technical direction documents
- `product/` — roadmap and product planning docs
- `plans/` — execution plans and implementation checklists
- `mockups/` — standalone prototypes and design artifacts

## Key docs

- `architecture/diagnosis-platform.md` — diagnosis-first product brief
- `architecture/source-to-diagnosis-flow.md` — long-term source, backend, finding, and evidence architecture
- `product/roadmap.md` — current phased roadmap
- `plans/README.md` — implementation plan index and status

## Code surfaces

- `../backend/` — backend app, module, and migration entrypoints
- `../frontend/` — product UI
- `../backend/internal/` — backend-owned internals
- `../backend/internal/metrics/` — backend-owned evidence and metric types
